# LitTCG — Full Task Plan: Make It Work and Make It Fun

> Comprehensive task list derived from code audit, doc audit, gap analysis, and the Tao of Fun design lens.
> Structured around the ADDIECRAPEYE workflow.
> **DO NOT edit this file with quick notes.** Update checkmarks and commit.

## How to use this plan

1. Work top-down within each phase. Do not open the next phase until the current phase can pass its `Definition of Done`.
2. Every task must include a verification step. If a task has no test, playtest, or `cargo` command, it is not done.
3. `cargo test` and `cargo clippy --features desktop` must pass after every phase.
4. Update `GDD.md`, `ARCHITECTURE.md`, `ROADMAP.md`, and `README.md` when systems change.

## Phase B: Design — Core Loop + Data Architecture (2–3 sessions)

**Goal:** Make the 60-second loop a *meaningful choice*, not a stat reveal.

### B.1 Combat Telegraph + Enemy Intent

- [ ] **B.1.1** Add `EnemyIntent` enum: `SynonymAttack`, `AntonymAttack`, `RootShield`, `GrammarShield`.
  - File: `src/core/battle.rs`.
  - Verify: unit test `enemy_intent_cycles` passes.
- [ ] **B.1.2** Each turn, the enemy broadcasts its next intent as a visible icon in `hud.rs`.
- [ ] **B.1.3** Player deals full counter damage only when playing the counter type; otherwise enemy attacks first or mitigates.
- [ ] **B.1.4** Combat log prints the reason, not just damage: e.g., `"Water EXTINGUISHES Fire — antonym counter!"`.
- [ ] **B.1.5** Add integration test: 5-turn battle with telegraph, counter, and enemy attack.

### B.2 Roster + Role Coverage

- [ ] **B.2.1** Design `Roster` resource: holds up to 6 pets, battle party of 3.
  - File: `src/core/components.rs` or new `src/core/roster.rs`.
- [ ] **B.2.2** Add `Role` bonuses in battle: Tank protects, Healer restores, Striker acts first, Caster breaks shields.
- [ ] **B.2.3** Pre-battle roster selection UI: pick 3 pets from collection.
- [ ] **B.2.4** Integration test: selecting a Tank reduces party damage taken.

### B.3 Face/Emotion Modifier System

- [ ] **B.3.1** Define `ActiveFace` resource and 4 face modes: Fierce, Joyful, Calm, Angry.
- [ ] **B.3.2** Apply face modifiers to `WordStats` before damage calc:
  - Fierce: +20% intensity, +10% dominance.
  - Joyful: +20% valence, +10% effective healing.
  - Calm: +20% concreteness, +10% defense.
  - Angry: +30% intensity, -10% valence, higher variance.
- [ ] **B.3.3** Show face modifier in combat log and on active-pet HUD.
- [ ] **B.3.4** Add visual transition between face icons.

### B.4 Damage Formula Visibility

- [ ] **B.4.1** Refactor `calculate_damage` to return a `DamageBreakdown` struct: base, synonym_bonus, antonym_bonus, face_bonus, role_bonus, mastery_bonus, final.
- [ ] **B.4.2** Render breakdown as color-coded lines in combat log.
- [ ] **B.4.3** Add unit test: breakdown sums to final damage.

### B.5 Data Cleanup

- [ ] **B.5.1** Remove `ArchetypeQuests` from `quest_data.json` if unused, or wire it through `QuestData`.
- [ ] **B.5.2** Add `archetype` field to `NpcData` if we keep archetype quests.
- [ ] **B.5.3** Validate that every NPC in `lore_db.json` has a quest chain and non-empty Night dialogue.
- [ ] **B.5.4** Add integration test: `start_quest("Nyx")` returns a valid quest template.

**Definition of Done:**
- [ ] A player can play a 1-minute battle, see enemy intent, choose a counter, and understand *why* they won or lost.
- [ ] `cargo test` passes with at least 5 new combat tests.
